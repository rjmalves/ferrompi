Expand description

Reduction operations

The Replace and NoOp variants are only available with the rma feature.

§Feature-gated variants

Without --features rma, referencing ReduceOp::Replace is a compile error:

// With --features rma, ReduceOp::Replace is available.
let _ = ferrompi::ReduceOp::Replace;

§

BitwiseOr = 4

Bitwise OR (MPI_BOR). Valid only for integer types; MPI returns MPI_ERR_OP when used with floating-point types.

§

BitwiseAnd = 5

Bitwise AND (MPI_BAND). Valid only for integer types; MPI returns MPI_ERR_OP when used with floating-point types.

§

BitwiseXor = 6

Bitwise XOR (MPI_BXOR). Valid only for integer types; MPI returns MPI_ERR_OP when used with floating-point types.

§

LogicalOr = 7

Logical OR (MPI_LOR). Interprets nonzero as true. Valid for integer types.

§

LogicalAnd = 8

Logical AND (MPI_LAND). Interprets nonzero as true. Valid for integer types.

§

LogicalXor = 9

Logical XOR (MPI_LXOR). Interprets nonzero as true. Valid for integer types.

§

MaxLoc = 10

Maximum value with location (MPI_MAXLOC). Returns the maximum value and the rank (index) where it occurred. Only valid with MpiIndexedDatatype via Communicator::allreduce_indexed.

§

MinLoc = 11

Minimum value with location (MPI_MINLOC). Returns the minimum value and the rank (index) where it occurred. Only valid with MpiIndexedDatatype via Communicator::allreduce_indexed.

§

Replace = 12

Replace the target buffer with the source value (MPI_REPLACE).

Only valid for MPI_Accumulate-family operations (Epic 7). Passing to allreduce, reduce, scan, etc. returns MPI_ERR_OP from MPI.

This variant is only present when the rma feature is enabled.

§

NoOp = 13

No-op: leaves the target buffer unchanged (MPI_NO_OP).

Only valid for MPI_Accumulate-family operations (Epic 7). Passing to allreduce, reduce, scan, etc. returns MPI_ERR_OP from MPI.

§Compile-time availability

This variant is only present when the rma feature is enabled.
